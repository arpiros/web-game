## 요약

현재 UI는 다크 판타지 톤과 기본 게임 흐름은 잘 잡혀 있다. 최근 공통 UI 클래스, 반응형 카드, 효과 칩, 진행도 바, 스킬 대상 선택 안내가 추가되면서 화면의 골격은 더 정돈되었다.

Playwright로 데스크톱 1440x900, 모바일 390x844 화면을 확인한 결과, 다음 단계에서는 “더 예쁘게”보다 “레이아웃이 실제 플레이 폭에서 안정적인가”를 먼저 봐야 한다. 특히 모바일 전투 화면은 현재 데스크톱 구조가 그대로 좁은 폭에 들어가면서 핵심 정보가 잘리고, 플레이 조작이 답답하게 보인다.

1. 모바일 전투 화면의 레이아웃 붕괴 수정
2. 데스크톱 전투 화면의 과도한 빈 공간과 작은 전투 대상 개선
3. 캐릭터 선택 화면의 카드 밀도/비교성 조정

5. 타이틀과 결과 화면의 인상 강화

## Playwright 스크린샷 기반 확인 결과

### 타이틀

- 데스크톱에서는 중앙 배치, 여백, 색감이 안정적이다.
- 모바일에서는 타이틀이 화면 폭을 거의 꽉 채워 임팩트는 좋지만, 글자가 지나치게 커서 게임 세계관이나 보조 정보가 들어갈 공간이 없다.
- 배경이 거의 추상 패턴뿐이라 첫 화면의 기억점은 약하다.

### 캐릭터 선택

- 데스크톱에서는 4장 + 1장으로 카드가 배치되어 마지막 카드가 다음 줄에 혼자 남는다. 넓은 화면에서 균형이 살짝 아쉽다.
- 모바일에서는 카드가 매우 크고 한 화면에 한 장 남짓만 보인다. 읽기는 좋지만 캐릭터 비교는 느리다.
- 스탯 막대는 유용하지만, 색상이 강해서 카드 내부의 다른 정보보다 더 먼저 튄다.
- 캐릭터 역할 태그가 없어서 “누가 어떤 플레이에 맞는지”는 여전히 텍스트를 읽어야 한다.

### 전투 데스크톱

- 중앙 전장이 지나치게 넓고 어둡다. 적 카드가 작고 하단에 놓여 있어 전투 대상의 존재감이 약하다.
- 파티 패널은 우측에 안정적으로 있지만, 정보가 작고 전장 대비 시각적 우선순위가 애매하다.
- 스킬 바는 작동 가능해 보이나 버튼과 전장 사이의 연결감이 약하다.
- 스킬 선택 후 대상 안내 배너와 타겟 표시가 잘 보인다. 이 개선은 유지할 가치가 있다.
- 스킬 tooltip이 대상 선택 중 왼쪽 하단에 뜨면서 전장과 겹친다. 위치 보정이 필요하다.

### 전투 모바일

- 가장 큰 문제 화면이다.
- 데스크톱의 가로 레이아웃이 그대로 유지되어, 전장과 파티 패널이 좌우로 찢어진다.
- 적 카드가 왼쪽 아래에 잘리고, 파티 패널이 화면 대부분을 차지한다.
- HUD 텍스트가 줄바꿈되어 “ROUND I / 30”, “플레이어 턴” 등이 너무 크게 보이고 공간을 많이 쓴다.
- 하단 스킬 바 버튼은 너무 커서 한 화면에 4개가 간신히 보인다. 장기적으로 스킬이 늘어나면 조작이 매우 힘들다.
- 모바일 전투는 별도 레이아웃을 반드시 설계해야 한다.

## 1. 전투 화면 개선

### 현재 아쉬운 점

- 전투 화면은 HUD, 적, 파티, 아이템, 시너지, 로그, 스킬 바가 한 화면에 모두 있어 정보량이 많다.
- 파티 패널이 항상 모든 정보를 보여주기 때문에 전투 중 핵심인 HP/MP/상태이상이 상대적으로 묻힐 수 있다.
- 스킬 버튼은 개선되었지만, MP 부족/쿨다운/대상 필요/전체 공격 여부가 아직 한눈에 완전히 구분되지는 않는다.
- 상태이상 배지는 존재하지만, 어떤 상태가 지금 가장 위험한지 우선순위가 약하다.
- 실제 데스크톱 화면에서는 전장이 너무 비어 보이고, 적 카드가 작아 전투의 중심이 약하다.
- 실제 모바일 화면에서는 전투 레이아웃이 깨져 플레이하기 어렵다.

### 제안

- 전투 화면을 세 영역으로 재정의한다.
  - 상단 HUD: 라운드, 턴, 현재 페이즈, 활성 시너지
  - 중앙 전장: 적, 의도, 타겟 상태, 데미지 피드백
  - 하단 조작: 스킬, 방어/턴 종료, 선택 안내
- 모바일에서는 완전히 다른 세로 레이아웃을 적용한다.
  - 상단: 압축 HUD
  - 다음: 파티 요약 가로 리스트
  - 중앙: 적 전장
  - 하단: 2열 스킬 그리드 또는 가로 스크롤 스킬 트레이
- 파티 패널은 “전투 판단용 요약”을 먼저 보여준다.
  - 캐릭터/동료 이름
  - HP/MP bar
  - 위험 상태이상 2~3개
  - 자세한 아이템/시너지는 접힘 또는 hover/detail로 이동
- 스킬 버튼에 상태별 시각 언어를 고정한다.
  - MP 부족: 붉은 MP 비용 + 낮은 opacity
  - 쿨다운: 회색 overlay + 남은 턴 크게 표시
  - 선택 중: 원소색 테두리 + 중앙 대상 선택 안내
  - 전체 공격: 버튼에 “전체” chip
- 적 의도(intent)를 더 명확히 한다.
  - 공격, 상태이상, 버프, 회복을 색상과 짧은 라벨로 분리
  - 다음 피해 예상치가 계산 가능하면 표시
- 데스크톱 전장은 적 카드 크기와 위치를 조정한다.
  - 적 카드를 현재보다 크게 표시
  - 적이 1마리일 때 중앙 하단이 아니라 중앙 중단에 배치
  - 적 수에 따라 카드 간격과 scale 조정
- tooltip은 화면 경계와 스킬 바 위치를 고려해 자동 배치한다.

## 2. 드래프트/조합 화면 개선

### 현재 아쉬운 점

- 카드가 보기 좋아졌지만, “이걸 왜 골라야 하는지”가 더 강하게 드러나면 좋다.
- 스킬/동료/아이템 카드의 정보 구조가 서로 조금 다르다.
- 조합 가능 레시피와 재료 부족 레시피가 길어지면 화면 피로도가 커질 수 있다.

### 제안

- 모든 드래프트 카드에 공통 하단 요약 영역을 둔다.
  - 스킬: 예상 피해/회복, MP, 쿨다운, 대상 유형
  - 동료: 행동 주기, 행동 유형, 예상 기여량
  - 아이템: 발동 조건, 적용 대상, 지속 효과 여부
- 카드 상단에는 선택 이유를 짧게 표시한다.
  - “화염 시너지 활성”
  - “생존력 보강”
  - “MP 순환 강화”
  - “보스전 대비”
- 조합 탭은 기본적으로 “가능한 조합”만 펼치고, “재료 부족”은 접힘 섹션으로 둔다.
- 회복 스킵은 현재 HP가 낮을 때 더 눈에 띄게 강조한다.
  - HP 35% 이하: 경고 스타일
  - HP 70% 이상: 낮은 강조

## 3. 캐릭터 선택 화면 개선

### 현재 아쉬운 점

- 스탯 막대와 초기 스킬 정보가 생겨 비교성은 좋아졌지만, 캐릭터의 플레이 스타일이 더 즉시 읽히면 좋다.
- 캐릭터별 시각적 상징이 아직 텍스트와 색상 중심이다.
- 데스크톱에서는 마지막 카드가 한 줄에 혼자 떨어져 배치 균형이 약하다.
- 모바일에서는 카드가 너무 커서 캐릭터 간 빠른 비교가 어렵다.

### 제안

- 캐릭터 카드에 역할 태그를 추가한다.
  - 예: 탱커, 폭딜, 회복, 속공, 하이리스크
- 카드별 “추천 플레이” 한 줄을 추가한다.
  - 예: “방어와 회복으로 길게 버티는 안정형”
- 원소별 문장 또는 간단한 아이콘을 추가한다.
  - SVG/아이콘 시스템으로 처리하면 이미지 에셋 없이도 충분하다.
- 선택 후 바로 시작하기보다, 선택된 캐릭터 상세 패널을 한 번 거치는 방식도 검토할 만하다.
- 데스크톱에서는 5개 캐릭터를 자연스럽게 보여주는 레이아웃을 선택한다.
  - 5열 고정에 가까운 넓은 그리드
  - 또는 선택 상세 패널 + 왼쪽 캐릭터 리스트
- 모바일에서는 카드 높이를 줄인 compact 모드를 적용한다.
  - lore는 2줄 clamp
  - 스탯은 주요 3개만 먼저 표시
  - 상세 정보는 펼침으로 이동

## 4. 이벤트 화면 개선

### 현재 아쉬운 점

- 이벤트 선택지는 효과 칩으로 좋아졌지만, 위험 선택과 안전 선택의 대비가 더 강해질 수 있다.
- 이벤트 자체의 분위기 이미지는 아직 부족하다.

### 제안

- 선택지마다 위험도 라벨을 붙인다.
  - 안전, 균형, 위험, 도박
- 패널 상단에 이벤트 유형을 표시한다.
  - 회복, 저주, 상인, 지식
- 이벤트별 배경 패턴 또는 작은 상징 아이콘을 둔다.
  - 야영지: 불씨
  - 저주 제단: 붉은 문양
  - 상인: 금화/저울
  - 고대 도서관: 책/룬

## 5. 결과 화면 개선

### 현재 아쉬운 점

- 통계와 보유 목록은 정리되었지만, 플레이어가 가장 보고 싶은 “이번 런의 하이라이트”가 약하다.
- 승리/패배 감정의 차이가 더 뚜렷해질 수 있다.

### 제안

- 상단에 하이라이트 3개를 먼저 보여준다.
  - 최고 단타
  - 가장 많이 사용한 스킬
  - 활성 시너지 수 또는 동료 수
- 승리/패배에 따라 배경 연출을 다르게 한다.
  - 승리: 금빛 선, 밝은 종결감
  - 패배: 낮은 채도, 붉은 잔광
- “같은 캐릭터로 재도전” 버튼을 추가한다.
  - 이를 위해 store에 마지막 선택 캐릭터를 기억하거나 result 상태에서 character id를 재사용하는 액션이 필요하다.

## 6. 모바일/반응형 개선

### 현재 아쉬운 점

- 카드 화면은 반응형이 좋아졌지만, 전투 화면은 여전히 데스크톱 중심이다.
- 전투 화면의 우측 파티 패널과 하단 스킬 바가 좁은 화면에서 동시에 부담이 될 수 있다.
- Playwright 모바일 스크린샷 기준, 전투 화면은 실제로 좌우 레이아웃이 깨져 적과 파티 영역이 동시에 읽히지 않는다.

### 제안

- 900px 이하에서는 전투 레이아웃을 세로형으로 반드시 전환한다.
  - 파티 패널을 우측 고정에서 상단/접힘 패널로 변경
  - 스킬 바는 2행 스크롤 또는 그리드로 변경
- 640px 이하에서는 전투 로그를 기본 접힘으로 고정한다.
- 터치 환경에서는 hover tooltip 대신 tap detail sheet를 사용한다.
- 모바일 HUD는 한 줄 압축을 원칙으로 한다.
  - `R 1/30`
  - `T 1`
  - `플레이어`
  - 속도는 아이콘 또는 작은 segmented control

## 7. 공통 UI/코드 구조 개선

### 현재 아쉬운 점

- 공통 CSS 클래스는 추가되었지만, 화면 컴포넌트에는 아직 인라인 스타일이 많다.
- `BattleScreen.tsx`는 파일이 크고 UI/애니메이션/계산 로직이 한 파일에 몰려 있다.

### 제안

- `src/components/ui/`를 만들고 반복 UI를 분리한다.
  - `Button`
  - `Card`
  - `Chip`
  - `ProgressBar`
  - `Tabs`
  - `Tooltip`
- `BattleScreen.tsx`를 기능 단위로 분리한다.
  - `BattleHud`
  - `EnemyZone`
  - `EnemyCard`
  - `PartyPanel`
  - `SkillBar`
  - `BattleLog`
- 계산성 UI 헬퍼는 별도 파일로 이동한다.
  - 예상 데미지 계산
  - 상태이상 라벨/색상
  - 원소 라벨/색상

## 검증 기준

- `npm run build` 통과
- `npm run test` 통과
- 변경 화면 대상 ESLint 에러 0개
- 데스크톱 1440x900, 노트북 1280x800, 모바일 390x844에서 주요 텍스트 겹침 없음
- 전투 중 다음 행동을 2초 안에 판단할 수 있는지 수동 확인
- 드래프트 카드 3장 비교 시 “선택 이유”가 카드마다 즉시 드러나는지 확인
- 모바일 전투 화면에서 적, 파티 HP, 스킬 버튼이 한 화면 안에서 모두 접근 가능해야 한다.
- 데스크톱 전투 화면에서 적 카드가 전장 중심 요소로 충분히 크게 보여야 한다.
