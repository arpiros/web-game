# UI/UX 개선 계획

작성일: 2026-04-26
대상: `src/App.tsx`, `src/screens/*`, `src/styles/*`

## 목표

- 전투 중 핵심 판단이 빠르게 되도록 정보 우선순위를 재정렬한다.
- 캐릭터 선택, 드래프트, 이벤트, 결과 화면의 선택지를 더 명확하고 보기 좋게 만든다.
- 현재의 다크 판타지 톤은 유지하되, 과한 장식보다 읽기 쉬운 구조와 절제된 디테일을 우선한다.
- 인라인 스타일 중심 구조를 한 번에 뒤엎지 않고, 공통 CSS 클래스와 토큰을 점진적으로 도입한다.

## 현재 상태 요약

- 화면 라우팅은 `src/App.tsx`에서 `run.phase` 기준으로 단순하게 분기되어 있다.
- 대부분 화면이 `style={{ ... }}` 인라인 스타일을 직접 사용한다.
- `src/styles/tokens.css`에는 OKLCH 기반 색상, 글꼴, 간격, 반경, 레이아웃 토큰이 이미 있다.
- `src/styles/ornaments.css`에는 장식 클래스가 있으나 실제 화면 적용은 제한적이다.
- `docs/reports/ui-review.md`에 과거 UI 점검 문서가 있고, 일부 항목은 이미 수정되어 있다.

## 주요 문제

1. 공통 컴포넌트/스타일 부재
   - 버튼, 카드, 배지, 패널, 섹션 헤더가 화면마다 직접 구현되어 일관성이 약하다.
   - hover 효과를 `onMouseEnter` / `onMouseLeave`로 처리하는 패턴이 많아 코드가 길고 유지보수가 어렵다.

2. 전투 화면 정보 밀도
   - `BattleScreen.tsx`가 매우 크고, HUD/적/파티/스킬/로그가 한 파일에 섞여 있다.
   - 적 카드와 스킬 버튼의 고정 크기가 많아 적 수, 긴 스킬명, 좁은 화면에서 답답해질 수 있다.
   - 스킬 선택 후 타겟 지정 상태는 동작하지만, 다음 행동이 무엇인지 더 강하게 알려줄 여지가 있다.

3. 선택 화면의 비교성 부족
   - 캐릭터 선택 화면은 숫자 스탯 나열 중심이라 캐릭터 간 장단점 비교가 느리다.
   - 드래프트 화면은 보상, 조합, 회복 스킵의 우선순위가 다소 분산되어 있다.
   - 이벤트 선택지는 효과 프리뷰가 작고, 위험/보상 차이가 시각적으로 즉시 들어오지 않는다.

4. 반응형과 접근성
   - `210px`, `140px`, `420px` 같은 고정값이 여러 화면에 있다.
   - 일부 버튼은 시각 텍스트에 의존하고, aria label/pressed/selected 상태가 부족하다.
   - 색상만으로 상태를 구분하는 곳이 있어 색약 사용자에게 불리할 수 있다.

5. 시각적 완성도
   - 타이틀, 결과, 이벤트 화면은 기능은 충분하지만 게임의 분위기를 대표하는 첫인상/마무리 연출이 약하다.
   - `--color-bg-card`처럼 정의되지 않은 토큰을 참조하는 코드가 있어 스타일 의도가 불명확하다.

## 개선 원칙

- 기능 추가보다 사용 흐름 개선을 먼저 한다.
- 전투 화면은 화려함보다 즉시 판단 가능한 배치를 우선한다.
- 카드 반경은 기존 토큰처럼 작게 유지하고, 패널 중첩은 피한다.
- 화면별 로컬 개선을 먼저 하고, 반복되는 패턴이 확인된 뒤 공통 클래스로 올린다.
- 게임 로직(`src/game`)은 건드리지 않고 UI/상태 구독층만 수정한다.

## 실행 계획

### 1단계: 스타일 기반 정리

- `tokens.css`에 누락 토큰을 보강한다.
  - `--color-bg-card` 사용 여부를 정리하거나 `--color-bg-surface`로 통합한다.
  - 긍정/부정 효과용 토큰을 추가한다. 예: buff, debuff, warning.
- `global.css` 또는 신규 `components.css`에 공통 클래스를 만든다.
  - `.ui-button`, `.ui-card`, `.ui-panel`, `.ui-chip`, `.ui-tabs`, `.ui-progress`
  - hover/focus/disabled 상태를 CSS로 이동한다.
- 전역 인라인 스타일 보정 셀렉터는 점진적으로 줄인다.

### 2단계: 전투 화면 UX 개선

- `BattleScreen.tsx`를 기능 단위로 분리한다.
  - HUD, EnemyZone, PartyPanel, SkillBar, BattleLog, Tooltip 계열을 별도 컴포넌트로 분리한다.
- 플레이어 턴 상태를 더 분명하게 만든다.
  - 스킬 선택 시 “대상 선택” 상태를 HUD 또는 적 영역 상단에 표시한다.
  - 타겟 가능한 적 카드에 동일한 리티클/테두리/커서 규칙을 적용한다.
- 스킬 바를 개선한다.
  - 긴 스킬명이 잘리지 않도록 2줄 표시 또는 tooltip을 제공한다.
  - MP 부족, 쿨다운, 타겟 필요 상태를 색상과 텍스트/아이콘 조합으로 구분한다.
  - 턴 종료 버튼은 스킬 목록과 같은 높이 체계로 맞추되, 별도 주요 액션으로 인식되게 한다.
- 파티 패널을 개선한다.
  - HP/MP 바와 상태이상 배지의 레이아웃을 안정화한다.
  - 동료, 아이템, 시너지 섹션을 접거나 밀도 높은 리스트로 정리할 수 있게 한다.
- 전투 로그는 기본 접힘/펼침 상태와 최신 로그 강조를 정리한다.

### 3단계: 캐릭터 선택 화면 개선

- 카드 그리드를 `repeat(auto-fit, minmax(...))` 기반으로 바꾼다.
- 캐릭터별 속성 색, 역할, 핵심 스킬을 한눈에 보이게 재배치한다.
- 스탯은 숫자와 막대 그래프를 함께 표시한다.
- 카드 hover/active/focus 상태를 CSS 클래스로 이동한다.
- 가능하면 캐릭터별 단순 실루엣/문장/원소 아이콘을 추가해 텍스트 의존도를 낮춘다.

### 4단계: 드래프트/조합 화면 개선

- 보상 선택, 조합, 회복 스킵의 우선순위를 명확히 한다.
  - 회복 스킵은 카드 아래 보조 옵션이 아니라 명확한 보조 액션으로 배치한다.
  - 현재 HP와 회복 후 예상 HP를 함께 보여준다.
- 드래프트 카드의 비교성을 높인다.
  - 스킬: MP, 쿨다운, 예상 피해/회복량, 원소
  - 동료: 행동 종류, 원소, 시너지 신규 활성 여부
  - 아이템: 효과 타입, 희귀도, 이미 보유 여부
- 조합 탭은 “가능한 조합”을 먼저, “재료 부족”은 접힘 또는 낮은 강조로 표시한다.
- 리롤 가능 상태와 남은 횟수를 카드 액션 영역에 직접 연결한다.

### 5단계: 이벤트/결과 화면 개선

- 이벤트 화면에 라운드 진행 정보와 선택지 위험도를 추가한다.
- 이벤트 선택지 효과는 작은 텍스트 대신 효과 칩으로 보여준다.
- 선택지 레이아웃은 grid 기반으로 바꿔 넓은 화면과 좁은 화면 모두 안정화한다.
- 결과 화면의 통계 패널 `minWidth`를 제거하고 반응형 그리드로 바꾼다.
- 결과 화면에 획득 아이템/스킬 희귀도 색상과 주요 성과 하이라이트를 추가한다.
- 같은 캐릭터로 재시작 기능은 별도 store 액션이 필요하므로 후순위로 검토한다.

### 6단계: 검증

- `npm run lint`
- `npm run build`
- `npm run test`
- 주요 화면 수동 확인
  - 타이틀
  - 캐릭터 선택
  - 전투 1라운드
  - 드래프트
  - 이벤트
  - 결과
- 가능하면 Playwright E2E 또는 스크린샷으로 1280px, 768px, 모바일 폭을 확인한다.

## 리스크와 주의점

- `BattleScreen.tsx`는 파일이 크고 상태/애니메이션이 얽혀 있으므로, 시각 개선과 컴포넌트 분리를 한 번에 크게 진행하면 회귀 위험이 있다.
- 게임 로직은 이미 순수 함수 계층으로 잘 분리되어 있으므로 UI 개선 중 `src/game` 변경은 피한다.
- 세리프 기반 폰트는 작은 크기에서 가독성이 떨어지므로, 수치/전투 정보는 mono 또는 더 큰 토큰을 유지한다.
- 장식 요소는 분위기를 보태는 정도로 제한하고, 버튼/카드의 클릭 가능성과 상태 인지가 흐려지지 않게 한다.
